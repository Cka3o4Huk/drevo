Well, while creating this Task, I realized that there is a really simple way to do this using the built in exec task. Below is an example script of how to build a very simple test.cpp file.

code: 

<?xml version="1.0"?>
<project name="mytask" default="gcc" basedir=".">
<target name="init">
	<property name="source" value="src/test.cpp"/>
	<property name="build" location="build"/>
</target>
<target name="gcc" depends="init">
	<mkdir dir="${build}"/>
	<exec dir="." executable="g++">
		<arg line="-o"/>
		<arg line="${build}/test"/>
		<arg line="${source}"/>
	</exec>
</target>
</project>

Here is the equivilent Makefile

code: 

cc = g++
objects = test.o
src = test.cpp
test : $(objects)
	$(cc) -o test $(objects)
	
$(objects) : $(src)
	$(cc) -c $(src)
	
all :
	$(MAKE) test


All g++ switches can be created in the property elements and then just referenced using the ${name} syntax. No big deal.. Much easier than writing a whole new task for this. Works like a champ.
